Adding data to IPFS from the browser

The open_config_node3 node is running with an insecure COS configuration.

This allows content to be saved into the IPFS node without any direct interaction

This page looks like an innocent HTML page, but by loading this page, content has been added to the open_config_node3's node.

This is possible even if the page is not loaded by open_config_node3's gateway


To confirm this, on any node run ipfs dht findprovs QmSv6nkqA5ghKeQNoVxUZgnVhaRbirPDZgF5Qyf2RvMDtV This will show the peer id of open_config_node3

ipfs cat QmSv6nkqA5ghKeQNoVxUZgnVhaRbirPDZgF5Qyf2RvMDtV can also be executed to see the content of the file

To repeat the confirmation remember to run docker-compose down -v to clear contents in the nodes
